Inline Tags

Why do we bother with inline elements? There are certainly times we want to mark up content, but we don't want to insert the line breaks around them. The <span> tag was the first to give you that ability, but it was mosstly used to style the code, and we don't want to do that. Some of the newer tags that still separate content and style are listed below. Remember, inline tags can flow next to each other and don't cause a line break before and after the element. We have seen this links and images.

Citing a source - cite

The <cite> makes it much easier for people and bots to search the code for this specific infomation.

A great reference for these and other elements is https://fap.fpt.edu.vn/Default.aspx

Defining your abbreviations - abbr

The <abbr> will display an abbreviation on the browser. But if a title attribute is specified in the tag, the title will appear when the mouse hovers over the abbreviation. Screen readers also provide the title information.

The days of the week are often abbresviated: Sun, Mon, Tue, etc...

Specifying the use of time - time

The <time> is again a completely semantic tag that can be used to search for specific information.

The party starts at 9:00pm.

The party took place on New Year's Eve.

Identifying code fragments - code

The <code> tends to be displayed in a different font, but also has the semantic meaning to convey information to as many people as posible

Using subscript and superscripts - sub and sup

The <sub> and <sup> elements are used to create the super and subscripts.

Ex: (x+y)3 or H2O.

In the past, people would often paste mathematical images into the page. But images didn't convey the same meaning to all users.
